## 2. 子项目 1：核心增强

### 2.1 港股支持

#### 设计目标

复用现有东方财富 API，支持港股数据获取，代码改动最小化。

#### 技术方案

| 组件 | 改动内容 |
|------|----------|
| 代码识别 | 港股代码为 5 位数字（如 00700），通过代码前缀或长度判断市场 |
| API 参数 | secid 格式改为 `116.XXXXX`（港股市场代码 116） |
| 价格单位 | 港股价格单位为毫（1/1000 港币），需除以 1000 |
| 名称映射 | 新增常见港股名称字典（腾讯控股、阿里巴巴、京东等） |

#### 代码结构

```python
# 市场判断逻辑
def get_market_info(code):
    """根据代码判断市场类型"""
    if len(code) == 5 and code.isdigit():
        return 'HK', 116  # 港股
    elif code.startswith('6'):
        return 'SH', 1    # 上海
    else:
        return 'SZ', 0    # 深圳

# 价格转换
def convert_price(raw_price, market):
    """根据市场类型转换价格"""
    if market == 'HK':
        return raw_price / 1000  # 港股：毫 → 港币
    else:
        return raw_price / 100   # A 股：分 → 人民币
```

#### K 线数据问题

测试发现港股 K 线接口不稳定，解决方案：
1. 增加重试次数（从 8 次增加到 12 次）
2. 增加超时时间（从 15 秒增加到 20 秒）
3. 失败时提示用户稍后重试

## 3. 子项目 2：分析能力扩展

### 3.1 估值分位数历史对比

#### 设计目标

实时计算当前估值在近 5 年历史中的百分位，帮助判断估值高低。

#### 技术方案

| 维度 | 计算方式 |
|------|----------|
| PE 分位数 | 当前 PE 在近 5 年 PE 序列中的百分位 |
| PB 分位数 | 当前 PB 在近 5 年 PB 序列中的百分位 |
| 股息率分位数 | 当前股息率在近 5 年股息率序列中的百分位 |

#### 数据获取

复用现有东方财富 API，获取近 5 年的估值数据：
- K 线数据（用于计算历史 PE/PB）
- 财务报表数据（用于计算历史 ROE、股息等）

#### 分位数计算

```python
def calculate_percentile(current_value, historical_values):
    """
    计算当前值在历史序列中的百分位
    返回: 0~100 的百分位数
    """
    if not historical_values:
        return None
    count_below = sum(1 for v in historical_values if v < current_value)
    return (count_below / len(historical_values)) * 100
```

#### 估值区间判断

| 分位数 | 区间 | 说明 |
|--------|------|------|
| 0%~20% | 低估 | 历史底部区域 |
| 20%~40% | 合理偏低 | 相对便宜 |
| 40%~60% | 合理 | 估值中位数 |
| 60%~80% | 合理偏高 | 相对贵 |
| 80%~100% | 高估 | 历史高位区域 |

#### 输出格式

```markdown
## 九、估值分位数分析

| 指标 | 当前值 | 5年分位数 | 估值区间 |
|------|--------|-----------|----------|
| PE(TTM) | 25.3 | 35.2% | 合理偏低 |
| PB | 3.8 | 42.1% | 合理 |
| 股息率 | 2.1% | 65.3% | 合理偏高 |

> 当前估值处于近 5 年 **合理偏低** 区间
```
